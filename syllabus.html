Syllabus for Project “Let’s Code!”

Unit 0 is delivered in workshop format during the first meet-up. It is expected to be completed during that session. Unit 5 is a “summary” unit that requires exposes students to more challenging and real-world problems that combine the knowledge gained from the various units. There is no lecture video/self-check quiz for unit 5, but there will be an assignment.

# Learning outcomes
0a Install Ruby on Windows on their own laptops.

0c Understand that you can run Ruby code in either interactive mode or ?normal? mode.
0d Understand basic concepts of programming, including what is a source file, how to load and run a Ruby program, and what are possible errors when a file is loaded (load time errors) or executed (runtime errors). Understand that statements in a source file are interpreted sequentially.

Unit 1
Unit # Name Recommended Start Date Recommended Completion Date
1 Variables, Types, Basic operations and Booleans 1st meet-up 2nd meet-up

At the end of the unit, participants should be able to:

# Description
1a Understand that variables are temporary storage spaces for values, and that you can assign a value to a variable using the = operator.
1b Understand that variables can store integers (whole numbers), floating point values (decimal numbers), strings and others.
1c Understand that constants are similar to variables in that they can store values, but unlike variables, their values cannot be altered.
1d Choose good variable names (nouns that start with a lower case letter, words are separated by underscores). Understand that there are reserved words in Ruby called keywords that cannot be used as variable names. Understand that a constant in Ruby has a name that starts with a capital letter.
1e Use the following mathematical operators with numbers and variables: + (addition),- (binary, subtraction), * (multiplication),/ (division), % (modulus), - (unary), ** (power to). Use the following operator on strings: + (concatenation). Understand the terms ?operand? and ?operator?.
1f Understand how to read the precedence table for operators.
1g Understand that if two operands are integers, the result is also an integer (e.g. 5/4 = 1, and not 1.2). On the other hand, if one of the operands is a float, the result will be a float.
1h Understand the difference between a string ("123") and an integer (123), and that mathematical operators can only work on integers. Understand that variables can store nil.
1i Use the following methods on variables to perform conversion to different types: .to_s(to string),to_i(to integer),to_f(to float).
1j Use common methods on string variables, such as .length, .downcase, .upcase, .strip, .delete(' ')
1k Understand the difference between normal and destructive methods (e.g. .downcasevs.downcase!)
1l Use puts/print and gets to print and obtain user input respectively.
1m Be able to combine the knowledge obtained so far to write very simple programs. (For example, participants should be able to write a program that prompts for two numbers and prints out the sum and difference of these two inputs.)
1n Understand what arebooleanvariables and that they can only store the values true or false.
1o Use the comparison operators: == (equals), != (not equals), , =. Understand that == (comparison) and = (assignment) are significantly different operators.
1p Use thebooleanoperators: ! (not), && (and), || (or). Understand the ANDandOR logical tables, and evaluate theoreticallybooleanexpressions involving a combination ofbooleanoperators.

2 Decisions and Loops 2nd meet-up 3rd meet-up

At the end of the unit, participants should be able to:

# Description
2a Use the if, else, else if, end keywords to write a decision statement. Understand the concept of a ?block? of statements.
2b Indent source code and understand why indentation matters.
2c Understand the concept of a loop and the sequence in which statements will be executed in a loop.
2d Write a loop using the (i) while keyword and (ii) for..in keyword. (All other looping constructs in Ruby will not be formally covered, but participants are encouraged to find out more.)
2e Write nested loops
2f Use the break and next keywords to prematurely terminate a loop
2g Be able to combine the knowledge obtained so far to write simple programs that involve conditionals and loops.
